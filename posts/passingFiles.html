TLDR

Start Python's Simple HTTP Server 


python3 -m http.server

Use Wget to retrieve file 


wget http://{IP}:{Port}/{file}

Introduction

Many a times while doing HackTheBox or even real work, I have needed a quick way to pass files around to different systems. In HackTheBox, I may send a reverse shell or some code from my machine to the target machine using this technique. While working I may need to quickly pass one log file to another machine. There are other ways to do this but most require some sort of login or ssh keys already added. Almost all systems, both Windows and Unix, have the Wget utility. Paring this with Python's HTTP server, we can easily pass files around that almost is never blocked due to the nature of the transfer. This is a classic tool that should be up any IT/Penetration Tester/SysAdmin sleeve.

Why this works

As I said, a lot of transfer methods require special privileges or have other requirements to succeed. This technique is almost never blocked because it uses the HTTP protocol. The only true way to totally block this kind of passing technique is to block HTTP. But that would grind most, if not all, businesses to the ground almost immediately as the entire internet relies on HTTP.

Furthermore, if you are trying to be stealthy, this is a fantastic method. In a real environment, there will be massive amounts of HTTP traffic being generated by everyday users doing their jobs, checking email, scrolling social medias, and wasting company time. Sure, a competent SysAdmin could cut a log of millions of lines down to a manageable number for analysis. But, HTTP is an innocuous protocol that should not set off any alarm bells.

How to do it

For this to work you must have some sort of Remote Code Execution on the target machine. This is because you either need to start the HTTP Server or send the command to use Wget. In our instance I will be showing all examples from one machine using localhost but this can work across any two computers that can talk to each other.

Let us imagine that there is a file that we wish to have on our machine named password.txt. First, you must be in the directory on the source machine where the item that you wish to pass is stored. Then you will start an HTTP Server using Python's Simple HTTP Server. 


$ python3 -m http.server

Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...

You will see that an HTTP server has been started on port 8000. This is the port that this module uses by default.

Now, on our destination machine we can use wget to retrieve the file that has been hosted by the HTTP server. Remember that we are trying to get the passwords.txt file. So we will use wget to download this. The syntax of of wget is as follows: wget http://{IP}:{Port}/file.name. 


$ wget http://127.0.0.1:8000/password.txt

--2021-02-03 00:19:27--  http://127.0.0.1:8000/password.txt

Connecting to 127.0.0.1:8000... connected.

HTTP request sent, awaiting response... 200 OK

Length: 18 [text/plain]

Saving to: ‘password.txt’

password.txt                                100%[==========================================================================================}]      18  --.-KB/s    in 0s

2021-02-03 00:19:27 (80.8 KB/s) - ‘password.txt’ saved [18/18]

In my instance, I am executing this all on my local machine so that is why I am using localhost but you can change the IP to whatever you need. As you can see, the password.txt file was downloaded and stored on the destination machines system.

Back on the source machine we can see the HTTP Get request that initiated the download. The first item is the IP address that downloaded the file, followed by the date and time of the request. The following field, in quotations marks, is the HTTP request. In our case, it was a Get request of the password.txt file. Finally there is the HTTP response code. In this case the response code was 200 which is OK. 


$ python3 -m http.server

Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...

127.0.0.1 - - [03/Feb/2021 00:19:27] "GET /password.txt HTTP/1.1" 200 -
